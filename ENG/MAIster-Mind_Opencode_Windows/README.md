# MAIster-Mind V1 (Windows Version via WSL)
*By Selim Boukhari* — [LinkedIn](https://www.linkedin.com/in/selim-boukhari-6356b949/)

**MAIster-Mind** is an automated code factory that drives the OpenCode AI agent via `tmux` in a WSL (Ubuntu) environment on Windows. Its purpose is to automate development (from functional requirements to final code) through a structured and human-validated pipeline, thus avoiding exclusive dependence on the most expensive remote models.

> 📦 For initial installation, see `INSTALL.md`.

---

## 🚀 Script Architecture

The factory offers 2 scripts depending on your target and goal (Code only vs Code + Tests):

| Script | Test Management |
|---|---|
| `MAIsterMind-code` | Production only |
| `MAIsterMind-test` | Production + Tests |

## 🛠️ Model Selection

The model is configured directly in OpenCode (via `/model` in the TUI or in `.opencode/opencode.json`).
* **Cloud:** Make sure you have configured your provider.
* **Local:** Launch Ollama in the background with the your model before running a local script.

## 📂 Minimal Project Structure

```text
├── need.md                      ← Precise functional brief (TO CREATE)
├── plan.md                      ← Sequential plan (Generated by AI)
├── blackboard.yaml              ← Phase tracking (Generated by AI)
├── refactoring_report.md        ← Final audit report (Generated by AI)
├── .agents/skills/              ← Technical guidelines by domain (architecture, frontend...)
├── .opencode/agents/factory.md  ← OpenCode automated agent configuration
├── MAIsterMind-code             ← Compiled orchestrator binary (CODE ONLY)
└── MAIsterMind-test             ← Compiled orchestrator binary (CODE + TESTS)
```

---

## 🛠️ The 4-Step Pipeline

Each script automatically executes the following steps:

1. **Planning:** Analysis of `need.md` and generation of `plan.md`.
2. **Blackboard:** Conversion of the plan into a structured YAML file (`blackboard.yaml`).
3. **Production:** Iterative code (and/or tests) loop with automatic criticism and retries.
4. **Refactoring:** Final audit, codebase cleanup and final report.

## 💻 Daily Usage

*All commands are executed in your WSL (Ubuntu) terminal, in the project directory.*

### 1. Prepare the need
Create and describe your functional requirement without ambiguity in `need.md`. Be extremely precise and indicate where to find files to reference.

### 2. Launch the orchestrator
Run the binary of your choice from the WSL terminal. Example for cloud with tests:
```bash
./MAIsterMind-test
```

### 3. Validate the Blackboard (Human-in-the-loop)
The script stops after step 2 and waits for your validation. You can:
* Type `y` to start production.
* Type `n` to stop everything (the tmux session will be killed).
* **Optional:** Directly modify the `blackboard.yaml` file in another terminal to adjust the AI phases before typing `y`.

### 4. Follow or Stop the AI
* **Follow live:** Open a second terminal and type:
  ```bash
  tmux attach -t oc-factory
  ```
  *(To exit the display without stopping the AI: Ctrl + B then D).*
* **Stop the process:** Simply do a `Ctrl + C` in the orchestrator terminal to properly close the session.

> ⚠️ **Important:** Do not let the produced code become a black box, always take the time to review it!

## 💡 Useful Commands & Troubleshooting

* **Monitor YAML progress:** `watch -n 2 cat blackboard.yaml`
* **Force AI shutdown:** `tmux kill-session -t oc-factory`
* **Resume after a crash:** Relaunch the binary, phases marked `DONE` will be automatically skipped.
* **Blocked on a confirmation:** Check that your `factory.md` agent has write permissions in `allow` mode.
