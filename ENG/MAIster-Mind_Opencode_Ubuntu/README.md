# MAIster-Mind V1 (Ubuntu / Debian Version - COMPILED BINARY)
*By Selim Boukhari* — [LinkedIn](https://www.linkedin.com/in/selim-boukhari-6356b949/)

**MAIster-Mind** is an automated code factory that drives the OpenCode AI agent via `tmux` directly on Ubuntu. Its purpose is to automate development (from functional requirements to final code) through a structured and human-validated pipeline, thus avoiding exclusive dependence on the most expensive remote models.

> 📦 See `INSTALL.md` for minimal setup.

---

## 🚀 Script Architecture

The factory offers 2 **compiled binaries** depending on your target and goal (Code only vs Code + Tests):

| Binary | Test Management | Command |
|---|---|---|
| `MAIsterMind-code` | Production only | `./MAIsterMind-code` |
| `MAIsterMind-test` | Production + Tests | `./MAIsterMind-test` |

## 💻 Daily Usage

*All commands are executed in your Ubuntu terminal, in the project directory.*

### 1. Prepare the need
Create and describe your functional requirement without ambiguity in `need.md`. Be extremely precise and indicate where to find files to reference.

### 2. Launch the orchestrator directly (no venv needed!)
Execute the compiled binary of your choice. Example for cloud with tests:
```bash
./MAIsterMind-test
```

### 3. Validate the Blackboard (Human-in-the-loop)
The script stops after step 2 and waits for your validation. You can:
* Type `y` to start production.
* Type `n` to stop everything (the tmux session will be killed).
* **Optional:** Directly modify the `blackboard.yaml` file in another terminal to adjust the AI phases before typing `y`.

### 4. Follow or Stop the AI
* **Follow live:** Open a second terminal and type:
  ```bash
  tmux attach -t oc-factory
  ```
  *(To exit the display without stopping the AI: Ctrl + B then D).*
* **Stop the process:** Simply do a `Ctrl + C` in the orchestrator terminal to properly close the session.

> ⚠️ **Important:** Do not let the produced code become a black box, always take the time to review it!

## 💡 Useful Commands & Troubleshooting

* **Monitor YAML progress:** `watch -n 2 cat blackboard.yaml`
* **Force AI shutdown:** `tmux kill-session -t oc-factory`
* **Resume after a crash:** Relaunch the binary, phases marked `DONE` will be automatically skipped.
* **Binary doesn't run?** Ensure you're in the correct directory and the binary has execute permissions.

## 🔒 Source Code Protection

This binary was compiled with **Nuitka** to protect the intellectual property of the orchestrator logic.
- The source code is **not accessible** from the binary
- All Python dependencies (PyYAML, etc.) are **included** in the binary
- The binary is **standalone** and does not require Python to be installed on the target system

## Customizing skills

You can add your own skills or modify existing ones. Each skill is a folder under `.agents/skills/<name>/SKILL.md` with YAML frontmatter (`name`, `description`) and behavioral instructions.

| Do | Don't |
|---|---|
| ✅ Add new skills (e.g., `python-django-api`, `docker-setup`) | ❌ Modify `plan/SKILL.md` or `plan-to-blackboard/SKILL.md` |
| ✅ Modify existing skills to match your stack | ❌ Remove or rename skills without updating the dictionary |

**When you add a skill**, register it in the `STRICT SKILLS DICTIONARY` inside `plan-to-blackboard/SKILL.md` by adding an entry like:

```yaml
"python-django-api":
  For implementing Django REST Framework views, serializers, models, admin config.
```

This is how the blackboard generator knows which skills to assign to each phase.
